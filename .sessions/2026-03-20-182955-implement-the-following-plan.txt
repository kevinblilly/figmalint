│                                                                              │
│ Context                                                                      │
│                                                                              │
│ FigmaLint currently supports Anthropic, OpenAI, and Google as AI providers   │
│ via a well-structured                                                        │
│ LLMProvider abstraction. The user wants to add AWS Bedrock so Claude models  │
│ can be accessed                                                              │
│ through AWS infrastructure (useful for enterprise licensing, compliance, and │
│  per-IAM-user                                                                │
│ credential distribution). Auth uses AWS Access Key ID + Secret Access Key +  │
│ Region rather than                                                           │
│ a single API key — and AWS Bedrock requires AWS Signature Version 4 (SigV4)  │
│ request signing,                                                             │
│ which is implemented using the browser's built-in crypto.subtle Web Crypto   │
│ API.                                                                         │
│                                                                              │
│ ---                                                                          │
│ Critical Design Decisions                                                    │
│                                                                              │
│ Credential Storage                                                           │
│                                                                              │
│ Bedrock requires 3 fields (accessKeyId, secretAccessKey, region) vs. a       │
│ single string for all                                                        │
│ other providers. These will be serialized as a JSON string and stored under  │
│ bedrock-api-key                                                              │
│ in Figma clientStorage — consistent with the existing ${providerId}-api-key  │
│ naming pattern.                                                              │
│                                                                              │
│ {"accessKeyId":"AKIAIOSFODNN7EXAMPLE","secretAccessKey":"wJalrXUtnFEMI/...", │
│ "region":"us-east-1"}                                                        │
│                                                                              │
│ SigV4 Signing                                                                │
│                                                                              │
│ callProvider() in index.ts currently calls provider.getHeaders(apiKey)       │
│ synchronously.                                                               │
│ SigV4 requires async crypto operations. Solution: mirror the existing Google │
│  special-case                                                                │
│ pattern in callProvider() with an if (providerId === 'bedrock') block that:  │
│ 1. Parses credentials from the JSON string                                   │
│ 2. Builds the dynamic endpoint URL                                           │
│ (https://bedrock-runtime.{region}.amazonaws.com/model/{modelId}/invoke)      │
│ 3. Calls signBedrockRequest() (async) to produce SigV4-signed headers        │
│ 4. Proceeds with the fetch using signed headers                              │
│                                                                              │
│ No interface changes needed.                                                 │
│                                                                              │
│ Request Format                                                               │
│                                                                              │
│ Bedrock's Claude invoke API uses the same Anthropic messages format with one │
│  difference:                                                                 │
│ - Replace anthropic-version header with anthropic_version:                   │
│ "bedrock-2023-05-31" in the body                                             │
│ - No model field in the body (model is in the URL path)                      │
│                                                                              │
│ Models (Claude on Bedrock only)                                              │
│                                                                              │

⏺ Here's a complete walkthrough. There are two separate things you need to set
  up in AWS — model access (in Bedrock) and IAM credentials (for
  authentication).

  ---
  Step 1: Enable Bedrock Model Access

  Bedrock models are off by default — this is the most common gotcha.

  1. Go to console.aws.amazon.com and sign in
  2. Switch to the region you want to use (e.g. us-east-1 — top-right dropdown)
  3. Search for "Bedrock" in the services search bar → open it
  4. In the left sidebar → "Model access"
  5. Click "Modify model access"
  6. Check the boxes for:
    - Claude 3.5 Sonnet v2 (anthropic.claude-3-5-sonnet-20241022-v2:0)
    - Claude 3.5 Haiku (optional)
    - Claude 3 Opus (optional)
  7. Click "Next" → "Submit"
  8. Wait for status to show "Access granted" (usually instant for Claude
  models)

  ▎ Important: Model access is per-region. If you use us-west-2, you must enable
   models in us-west-2.

  ---
  Step 2: Create an IAM User with Bedrock Permissions

  1. Go to IAM → Users → Create user
  2. Give it a name (e.g. figmalint-bedrock)
  3. Permissions step → choose "Attach policies directly"
  4. Search for and attach AmazonBedrockFullAccess
    - Or for least-privilege, create a custom policy:
    {
    "Version": "2012-10-17",
    "Statement": [
      {
        "Effect": "Allow",
        "Action": "bedrock:InvokeModel",
        "Resource": "arn:aws:bedrock:*::foundation-model/anthropic.*"
      }
    ]
  }
  5. Click through to create the user

  ---
  Step 3: Generate Access Keys

  1. Open the user you just created → "Security credentials" tab
  2. Scroll to "Access keys" → "Create access key"
  3. Use case: select "Other" (or "Application running outside AWS")
  4. Click through → you'll see:
    - Access key ID — looks like AKIAIOSFODNN7EXAMPLE
    - Secret access key — shown only once, copy it now
  5. Download the CSV or copy both values

  ---
  Step 4: Enter in the Plugin

  In the FigmaLint settings with AWS Bedrock selected:
  - Region: match exactly what you enabled models in (e.g. us-east-1)
  - Access Key ID: the AKIA... value
  - Secret Access Key: the long secret value

  Quick Validation Test

  Before trying the plugin, you can verify credentials work via AWS CLI (if you
  have it):

  aws bedrock-runtime invoke-model \
    --region us-east-1 \
    --model-id anthropic.claude-3-5-sonnet-20241022-v2:0 \

  — and if the plugin fails at that point it's a code issue. If the CLI also
  fails, it's a credentials/permissions issue.

  1. Model access not enabled in the right region
  2. Wrong region selected in the plugin vs where models are enabled
  3. IAM user missing bedrock:InvokeModel permission
  4. Copying the access key ID vs secret mixed up (secret is the much longer
